# Plain Language Checker

Review and improve documents for clarity using universal plain language principles.

## Overview

Analyzes documents against plain language guidelines to create clearer writing. Based on SEC's "A Plain English Handbook," but applies universally across languages—including English, Chinese (简明中文), and others.

**Reference:** See @references/sec-handbook-reference.md for detailed examples.

## When to Use

- Reviewing any document for clarity and readability
- Editing legal, financial, or technical documents for general audiences
- Converting complex documents into plain language
- Training writers on plain language principles
- Preparing disclosure documents, reports, or communications

## When NOT to Use

- Creative writing, poetry, fiction (intentional stylization)
- Legal documents requiring specific phrasing for compliance
- Historical documents preserving original voice
- Source code comments needing technical precision

## Quick Reference: The Core Principles

| #   | Principle | Quick Check | Examples (EN / 中文) |
| --- | --------- | ----------- | -------------------- |
| 1   | Active voice | "is/are/was/were" + past verb + "by" / "被..." | "The stock is bought by the investor" / "股票被投资者购买" → "投资者购买股票" |
| 2   | Short sentences | Average 15-20 words / 15-25字 | Break long sentences; one idea per sentence |
| 3   | Personal pronouns | "we/us/our" "you/your" / "我们/你/您" | "We recommend..." / "我们建议..." instead of passive constructions |
| 4   | Concrete terms | Replace abstractions | "One share of Apple" / "一股苹果股票" not "equity investment vehicle" |
| 5   | Omit superfluous words | "in order to" → "to" / "为了...起见" → "为了" | Remove words that don't add meaning |
| 6   | Write positively | "not able" → "unable" / "不是没有" → "有" | Positive sentences are shorter and clearer |
| 7   | Replace jargon | "terminate" → "end" / "终止" → "结束" | Use common words; explain technical terms |
| 8   | Natural word order | Keep subject-verb-object together | "The committee approved the proposal" / "委员会批准了提案" |
| 9   | If-then conditionals | One if → one then / 如果...那么... | Tabulate complex conditionals |
| 10  | Parallel structure | Lists use same grammatical form | "to provide, to protect, to generate" / "提供...保护...生成..." |
| 11  | Avoid "respectively" | State relationships explicitly / 分别→具体说明 | Don't force readers to match items |
| 12  | **Spell it out literally** | Don't hint with big words / 不要拐弯抹角 | Say exactly what you mean / 直说 |
| 13  | **Brevity above cleverness** | Simple beats fancy / 简洁胜过花哨 | Write to inform, not impress / 传达信息，不是炫耀 |

## How to Use

| Step | Action |
|------|--------|
| 1 | Read once for overall flow |
| 2 | Analyze: Check against core principles |
| 3 | Rewrite problematic sections |
| 4 | Output per format below |

**Quick Checklist:**
- Sentence: Active voice? Under 20 words? Natural order?
- Words: Strong verbs? No jargon? Concrete terms? Personal pronouns?
- Organization: Big picture first? Descriptive headers? Lists for complex info?

**For each issue:** Quote original → Explain → Rewrite → Cite principle

See @references/sec-handbook-reference.md for detailed examples.

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Only fixing grammar | Focus on reader understanding, not just correctness |
| Over-simplifying jargon | Explain necessary terms; don't eliminate them |
| Rewriting without examples | Always show before/after comparisons |
| Skipping the checklist | Systematic review catches more issues |

## Output Format

| Section | Content |
|---------|---------|
| Summary | Document readability overview |
| Key Findings | Issue category + original quote + problem explanation + rewritten version + principle violated |
| General Recommendations | Overarching suggestions as bullets |
| Positive Observations | What the document does well |

**For each issue:** Quote original → Explain problem → Provide rewrite → Reference principle.

## Red Flags - STOP and Check

| Excuse | Reality |
|--------|---------|
| "Big words show expertise" | Clear words communicate. |
| "This sounds more impressive" | Fancy writing distracts from the message. |
| "I need to soften subtly" | Hinting confuses. State literally. |
| "Formal documents must sound official" | Formal ≠ unclear. Plain language works for legal docs. |
| "Audience understands jargon" | Experts prefer clarity. Jargon excludes. |
| "Might alter legal meaning" | Check with legal, but most can be clearer. |
| "Just a style preference" | Backed by readability research. |
| "Too time-consuming" | Focus on high-impact changes first. |
| "Academic tone needs complexity" | Best academic writing is clear. Complexity ≠ intelligence. |

## Important Notes

- **Don't create boilerplate**: Clearer communication, not new templates
- **Context matters**: Explain necessary technical/legal terms; don't eliminate them
- **Audience awareness**: Adjust complexity for your readers
- **Substance first**: Clarity enhances, not replaces, accurate information

## Cross-Language Application

| Language | Notes |
|----------|-------|
| English | Principles apply directly |
| Chinese (中文) | 主谓宾结构清晰; 避免长句套从句; 减少文言残留 |
| Others | Focus: active voice, short sentences, personal tone, concrete examples |
